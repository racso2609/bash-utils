# MongoDB Bulk Export & Import Utilities 🚀

Here's the thing: moving data between MongoDB databases shouldn't be a manual, tedious process. These bash scripts automate the extraction of all collections from a database into a structured folder, and allow you to seamlessly import them into a new database. Solid foundations!

## 🛠 Dependencies

Before you run anything, you need the right tools. I see people trying to run these without the binaries and getting frustrated. Don't be that guy! You need:

1. **`mongosh` (MongoDB Shell):** Required by the export script to dynamically query the database name and the list of collections.
2. **`mongoexport` & `mongoimport` (MongoDB Database Tools):** The heavy lifters that actually read and write the BSON/JSON data.

### How to install them:
- **Ubuntu/Debian:** `sudo apt-get install mongodb-mongosh mongodb-org-tools`
- **macOS (Homebrew):** `brew tap mongodb/brew && brew install mongosh mongodb-database-tools`
- **Windows:** Download the MSI installers directly from the MongoDB website.

## 📦 1. Exporting Data (`mongo_export_all.sh`)

This script connects to your source database, figures out its name, reads all available collections (skipping internal `system.*` collections), and exports each one as a JSON file.

It creates a directory named `<db_name>-export` and saves files inside it using the format: `[db_name-collection_name].json`.

### Usage:
```bash
./mongo_export_all.sh <MONGO_URI> <PLACEHOLDER_NAME>
```

### Example:
```bash
./mongo_export_all.sh "mongodb://user:pass@localhost:27017/my_legacy_db" "Backup-Before-Migration"
```
**Result:** You get a folder named `my_legacy_db-export/` containing files like `[my_legacy_db-users].json`, `[my_legacy_db-products].json`, etc.

---

## 📥 2. Importing Data (`mongo_import_all.sh`)

This script takes the folder generated by the export script and pushes it to a new MongoDB URI. It smartly strips the database prefix from the filenames so your data goes into the correct collection names (e.g., `users`, not `my_legacy_db-users`).

*Note: It uses `--mode=upsert` under the hood. If a document already exists with the same `_id`, it updates it instead of crashing. It's that simple.*

### Usage:
```bash
./mongo_import_all.sh <NEW_MONGO_URI> <EXPORT_FOLDER>
```

### Example:
```bash
./mongo_import_all.sh "mongodb://user:pass@remote-server:27017/my_new_db" "my_legacy_db-export"
```
**Result:** All JSON files inside `my_legacy_db-export/` are imported into `my_new_db`.

---

## ⚠️ Important Notes
- Always wrap your connection URIs in quotes (`" "`) in the terminal to prevent bash from misinterpreting special characters like `&` or `?`.
- Ensure the scripts are executable before running them: `chmod +x *.sh`
